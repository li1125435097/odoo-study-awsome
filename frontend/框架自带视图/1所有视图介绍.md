在 **Odoo 17** 中，框架原生支持多种**视图类型（View Types）**，用于以不同方式展示和交互模型数据。这些视图类型由 Odoo 的前端（基于 OWL 框架）和后端共同定义，每种都有特定的 XML 结构、属性和用途。

以下是 **Odoo 17 所有官方支持的视图类型**（包括社区版和企业版），并标注其可用范围：

---

### ✅ Odoo 17 官方视图类型总览

| 视图类型（XML 标签名） | 社区版 (Community) | 企业版 (Enterprise) | 主要用途 |
|------------------------|--------------------|---------------------|--------|
| `<form>`               | ✅                 | ✅                  | 单记录编辑/查看（表单） |
| `<list>`（或 `<tree>`）| ✅                 | ✅                  | 多记录表格列表（可内联编辑） |
| `<kanban>`             | ✅                 | ✅                  | 卡片式看板（支持拖拽分组） |
| `<graph>`              | ✅                 | ✅                  | 数据可视化（柱状图、折线图、饼图） |
| `<pivot>`              | ✅                 | ✅                  | 透视表（多维交叉分析） |
| `<calendar>`           | ✅                 | ✅                  | 日历视图（按日/周/月展示时间事件） |
| `<gantt>`              | ❌                 | ✅                  | 甘特图（项目进度计划） |
| `<dashboard>`          | ❌                 | ✅                  | 仪表盘（组合多个子视图） |
| `<search>`             | ✅                 | ✅                  | 搜索/过滤/分组控件（非独立视图，配合其他视图使用） |
| `<qweb>`               | ✅                 | ✅                  | 自定义 HTML 模板（常用于报表、邮件、自定义页面） |

> 📝 注：
> - `<tree>` 是 `<list>` 的旧别名，Odoo 16+ 推荐使用 `<list>`，但两者完全兼容。
> - `<search>` 不是独立视图，而是为其他视图提供搜索栏功能。
> - `<qweb>` 通常不作为主视图类型注册，而是用于 `ir.ui.view` 的 `type="qweb"` 或在 JS 中渲染。

---

### 🔍 各视图类型详解

#### 1. **`<form>`**
- **用途**：显示和编辑单条记录。
- **特点**：支持字段、按钮、notebook（标签页）、group、sheet 等布局。
- **示例**：
  ```xml
  <form>
      <sheet>
          <group>
              <field name="name"/>
              <field name="email"/>
          </group>
      </sheet>
  </form>
  ```

#### 2. **`<list>`（或 `<tree>`）**
- **用途**：表格形式展示记录列表。
- **特点**：支持排序、分组、内联编辑、行装饰（`decoration-*`）。
- **示例**：
  ```xml
  <list editable="top">
      <field name="name"/>
      <field name="state"/>
  </list>
  ```

#### 3. **`<kanban>`**
- **用途**：卡片式看板，适合状态流管理（如任务、机会）。
- **特点**：支持拖拽、分组、自定义 QWeb 模板、颜色标记。
- **示例**：
  ```xml
  <kanban default_group_by="stage_id">
      <field name="name"/>
      <templates>
          <t t-name="kanban-box">
              <div><t t-esc="record.name.value"/></div>
          </t>
      </templates>
  </kanban>
  ```

#### 4. **`<graph>`**
- **用途**：数据聚合可视化。
- **类型**：`bar`、`line`、`pie`、`pivot`（注意：`pivot` 类型实际应使用 `<pivot>` 视图）。
- **示例**：
  ```xml
  <graph type="bar">
      <field name="date" type="row"/>
      <field name="amount" type="measure"/>
  </graph>
  ```

#### 5. **`<pivot>`**
- **用途**：多维度交叉分析表（类似 Excel 透视表）。
- **特点**：行/列/指标自由拖拽。
- **示例**：
  ```xml
  <pivot>
      <field name="product_id" type="row"/>
      <field name="partner_id" type="col"/>
      <field name="quantity" type="measure"/>
  </pivot>
  ```

#### 6. **`<calendar>`**
- **用途**：按时间轴展示事件。
- **必需属性**：`date_start`，可选 `date_stop`、`color`、`all_day`。
- **示例**：
  ```xml
  <calendar date_start="start_date" color="user_id">
      <field name="name"/>
      <field name="user_id"/>
  </calendar>
  ```

#### 7. **`<gantt>`**（仅 Enterprise）
- **用途**：项目甘特图，展示任务时间线和依赖关系。
- **依赖字段**：`start_date`、`stop_date`、`progress`、`dependency_task_ids`。
- **示例**：
  ```xml
  <gantt 
      date_start="start_date"
      date_stop="end_date"
      progress="progress"
      dependency_arrow="dependency_task_ids"
      class="o_project_gantt"
  >
      <field name="name"/>
      <field name="user_id"/>
  </gantt>
  ```

#### 8. **`<dashboard>`**（仅 Enterprise）
- **用途**：组合多个子视图（graph/list/kanban 等）到一个页面。
- **结构**：包含多个 `<view type="..." model="...">` 子节点。
- **示例**：
  ```xml
  <dashboard>
      <view type="graph" model="sale.order">...</view>
      <view type="list" model="project.task">...</view>
  </dashboard>
  ```

#### 9. **`<search>`**
- **用途**：定义搜索过滤器、分组选项、扩展筛选器。
- **非独立视图**：自动附加到其他视图的搜索栏。
- **示例**：
  ```xml
  <search>
      <field name="name"/>
      <filter name="my_tasks" string="My Tasks" domain="[('user_id', '=', uid)]"/>
      <group expand="0" string="Group By">
          <filter name="by_stage" string="Stage" context="{'group_by': 'stage_id'}"/>
      </group>
  </search>
  ```

#### 10. **`<qweb>`**
- **用途**：纯 HTML 模板，用于自定义页面、报表、邮件模板等。
- **注册方式**：`<template id="xxx">` 或 `ir.ui.view` with `type="qweb"`。
- **示例**：
  ```xml
  <template id="my_custom_page">
      <div class="custom-dashboard">
          <h1>Welcome</h1>
      </div>
  </template>
  ```

### 🧩 特殊说明

- **`<activity>` 视图？**  
  Odoo 没有独立的 `<activity>` 视图类型。活动（Activities）通常集成在 `<form>` 或 `<kanban>` 中通过 `mail.activity.mixin` 实现。

- **`<cohort>` 视图？**  
  Odoo 曾有 Cohort（同期群）视图，但在 **Odoo 15+ 已移除**，不再支持。

- **自定义视图类型**：  
  开发者可通过继承 `View` 和注册新 `type` 创建自定义视图（需实现 JS 控制器和 OWL 组件）。
